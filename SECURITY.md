# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability, exposed
credential, or privacy problem.

Use GitHub's private vulnerability reporting flow from the repository's
**Security** tab. If private reporting is not available, contact the repository
owner privately through their GitHub profile and include only the minimum
information needed to reproduce the issue.

Do not include live credentials, private datasets, personal information, or
unredacted logs in a report.

## Supported version

Security fixes are applied to the latest revision on the default branch.

## Scope

EXOHUNT is local research software. The dashboard binds to `127.0.0.1` and is
not designed to be exposed directly to the internet. Reports about unintended
network exposure, unsafe file access, dependency vulnerabilities, or credential
handling are welcome.
